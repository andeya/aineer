> 状态：提案草案

## 一、现状分析

### 当前技术栈

- **框架**：eframe + egui (即时模式 UI)，wgpu 后端
- **终端**：fork 的 egui_term，基于 alacritty_terminal
- **主题**：完成了 Dark/Light 双主题 Palette 系统（`ThemeMode` + `thread_local` 调色板）

### 已解决的问题（本次迭代）

1. ✅ 单例进程检测修复（先 connect 再 bind）
2. ✅ Dark/Light 主题切换（Palette 架构 + 即时预览）
3. ✅ 输入栏 Shell 化（显示 CWD、git 分支）
4. ✅ Diff 面板随 cd 更新 git watcher
5. ✅ WIN CI clippy 错误

### 待改进的核心问题

1. **Settings UI 过于简陋** — 缺少分组卡片、图标、描述文本
2. **Shell 终端分屏** — 目前是固定分割，不支持多终端分屏
3. **对比度/可读性** — 部分 UI 元素在不同主题下对比度不佳
4. **缺少动画/过渡** — 切换面板、展开/折叠没有动画
5. **响应式布局** — 窗口过小时 UI 挤压

---

## 二、设计目标

对标 VS Code 的设计哲学：

1. **清晰的信息层次** — 通过间距、字重、颜色建立视觉层级
2. **一致的交互模式** — 所有按钮、输入框、下拉菜单遵循统一设计语言
3. **高效的空间利用** — 面板可折叠、可拖拽、可自定义
4. **Professional 质感** — 精细的阴影、边框、圆角、图标

#### 1.2 基础组件库（`crates/ui/src/widgets/`）

- [ ] `IconButton` — 带图标的按钮，支持 tooltip
- [ ] `SectionHeader` — 分组标题栏（带折叠、图标、操作按钮）
- [ ] `SettingsRow` — 设置项行（标签 + 控件 + 描述）
- [ ] `StatusBadge` — 状态指示器（运行中/成功/失败/警告）
- [ ] `TabBar` — 统一的标签栏组件（支持拖拽排序、关闭、中键关闭）
- [ ] `SplitPane` — 可拖拽分割面板（支持水平/垂直）
- [ ] `Breadcrumb` — 路径面包屑导航
- [ ] `Toast` — 轻量级消息提示
- [ ] `ContextMenu` — 右键菜单
- [ ] `EmptyState` — 空状态占位（图标 + 描述 + 操作）

#### 1.3 图标系统

- 引入 SVG 图标集（推荐 codicons 或 lucide，通过 `egui::Image` + `include_bytes!`）
- 替代当前 Unicode 符号（⚙、✕、⎇ 等）为矢量图标
- 建立图标注册表 `ui::icons::ICON_NAME`

## 六、验收标准

1. **视觉一致性**：所有组件遵循设计令牌，无颜色硬编码
2. **主题完整性**：Dark/Light 主题下所有 UI 元素可读且美观
3. **响应式**：在 800×600 到 4K 分辨率下均能正常使用
4. **键盘友好**：所有核心操作可通过键盘完成
5. **性能**：60fps 流畅渲染，Timeline 10000+ 卡片无卡顿
6. **对比度**：WCAG AA 级别（4.5:1 正文，3:1 大字）
